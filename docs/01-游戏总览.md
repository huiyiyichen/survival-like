# 01 游戏总览

## 项目定义
本项目是一款面向 **PC** 键鼠平台的单人生存动作游戏，玩法形态接近吸血鬼幸存者。玩家通过移动走位、自动攻击、击杀拾取经验与升级构筑，在一张主地图内完成 12 分钟的生存与终局决战。整体美术统一采用像素风，并通过高对比轮廓和高亮技能特效保证战斗识别度。

题材采用奇幻魔物入侵。世界观只承担解释敌人、地图和角色来源的职责，不作为主驱动力，避免课程方案被剧情稀释。

## 设计目标
首版目标不是做内容堆砌，而是做出一个闭环完整、容易讲清、便于演示的生存流程。课程汇报时，应能用一张流程图、一条时间轴和几张配置表说明游戏如何运行。

| 目标 | 说明 | 验收方式 |
| --- | --- | --- |
| 流程完整 | 从主菜单到结算再到局外解锁形成闭环 | 单次演示能完整跑通 |
| 反馈清晰 | 升级、宝箱、Boss、解锁都有明确提示 | 关键节点不会被忽略 |
| 内容适中 | 小而完整，不依赖大量数值和剧情填充 | 3 名角色、1 张地图即可支撑方案 |

## 体验支柱
这款游戏的体验重心落在“短局高密度成长”。一局时间不长，但每个阶段都需要给出新压力或新回报，避免中段空转。

1. 开局快速进入状态，前 3 分钟完成规则教学。
2. 中段通过精英、Boss 和宝箱拉升节奏，促使构筑成型。
3. 终局必须有明显高潮，倒计时结束后仍保留最终收尾压力。
4. 失败后仍能推进局外解锁，保证下一局有明确动力。

## 版本范围
首版只实现一个核心生存模式，不加入无尽模式、剧情章节或多人联机。内容规模控制在课程展示可解释、可维护的范围内。

| 模块 | 首版包含 | 首版不做 |
| --- | --- | --- |
| 模式 | 单人核心生存模式 | 无尽模式、挑战模式 |
| 地图 | 1 张主地图“诅咒林地” | 区域推进、多地图章节 |
| 成长 | 4 武器 + 4 被动构筑、武器进化、局外解锁 | 永久属性商店、复杂天赋树 |
| 内容量 | 3 名角色、6 武器、6 被动、4 普通敌人、2 精英、1 终Boss | 大量剧情文本和扩展职业线 |

## 成功标准
项目成功与否，不看素材数量，而看核心流程是否自洽。只要以下条件成立，这套课程方案就具备继续实现或汇报的基础。

| 维度 | 标准 |
| --- | --- |
| 新手可懂 | 第一次进入能在 1 分钟内理解基本玩法 |
| 节奏可讲 | 12 分钟时间轴中每个阶段都能说明事件与目的 |
| 系统可拆 | 角色、武器、敌人、界面能分文档维护 |
| 后续可扩 | 新增角色或武器时，不需要推翻主流程 |
